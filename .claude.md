# Secretary 프로젝트 - Claude 작업 지침

## 프로젝트 개요
여러 AI(Claude, Gemini, Grok, GPT)와의 대화를 통합 관리하고, AI 간 맥락을 공유하는 시스템

## 기술 스택
- **프론트엔드**: Next.js 16 + TypeScript + Tailwind CSS
- **백엔드/DB**: Supabase (PostgreSQL)
- **배포**: Vercel

## 프로젝트 구조
```
src/
├── app/           # Next.js 앱 라우터
├── components/    # UI 컴포넌트
│   ├── Dashboard.tsx       # 오늘의 현황
│   ├── SecretarySection.tsx # 비서 섹션
│   ├── SearchSection.tsx   # 검색 & 맥락 프롬프트
│   ├── FileUpload.tsx      # 파일 업로드
│   ├── ExpertLinks.tsx     # 전문가 바로가기
│   └── ArchiveSection.tsx  # 일일 아카이브
├── lib/           # 유틸리티
│   └── supabase.ts         # Supabase 클라이언트
└── types/         # TypeScript 타입 정의
    └── database.ts
```

## Supabase 테이블
- **conversations**: 대화 기록 (id, ai_source, role, title, content, summary, tags)
- **daily_archives**: 일일 아카이브 (id, date, summary, todos, ideas, timeline)

## 코딩 규칙
- 한국어 주석 사용
- 컴포넌트는 함수형 + TypeScript
- Tailwind CSS 클래스 순서: layout → spacing → typography → colors
- `any` 타입 사용 금지 - 반드시 타입 정의

## 필수 사항
- Supabase 쿼리 시 **반드시 에러 핸들링** 할 것
- 환경변수는 `NEXT_PUBLIC_` 접두사 필수 (클라이언트 사용 시)
- 파일 수정 후 TypeScript 에러 확인
